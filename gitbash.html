gitbash

what is gitbash?

Git Bash is an application for Microsoft Windows environments which provides an emulation layer for a Git command line experience. Bash is an acronym for Bourne Again Shell. A shell is a terminal application used to interface with an operating system through written commands.

gitbash structure

Repository

The overall package or project. There is typically a remote of origin respository, and a local copy of it at certain time on your local machine

Branch

An extension of a repository. All repositories, or repos,start with a 'master' branch. From there, new branches can be created and used for new features. The idea is to keep the master branch clean and working for the final " done" project.

Commit

a commit is a change to the repository. For every commit made, there is a recod of the state of the repository at that given point in time.

cd (change directory)

cd

cd ..

moves up one directory

ls

lists the files or directories

git init

creates a new local respository in the current directory

git cheakout -b {branch name}

shorthand comman to create and switch to a new branch

git branch {branch name}

creates a new branch

git cheakout {branch name}

switches to the branch

git branch -a

displays all branches, both local and remote, that are available

git status RUN A LOT

Tells what files have been added, deleted, and modified Informs if the local branch is behind the remote version or if the version is ahead of the remote repository

git add {file, directory, or, other}

stages the changes to be committed

git branch .

stages all changes from the directory

git commit -m"" (commit message)

This commits the added changes locally and attaches the commit message to the commit. commit messages chould be short but decriptive of all the changes made. committing dose not push changes to the remote repository

git push

takes the locally committed changes for a given branch and applies them to the local version

git push -u origin {branch name}

This command is used for pushing to the remote repository with the specific branch name

git pull

takes the changes made to the remote repository and applies them to the local version
